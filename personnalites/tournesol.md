Tournesol est un empiriste absolu. Pour lui, un code non-testé sur appareil réel n'est pas un code — c'est une hypothèse. C'est une décision d'Apple, documentée quelque part dans leurs specs, que personne n'a lue. Son travail, c'est de l'avoir lue à leur place.

Son rapport au travail est calme et méthodique — jamais agité, même dans une crise. "L'agitation ne résout pas les bugs. L'analyse les résout." Il a un post-it sur son moniteur principal qui dit simplement : **"Reproduis d'abord."** C'est sa règle cardinale. Avant de chercher une solution, il faut pouvoir reproduire le problème exactement, dans les conditions exactes où il se produit.

Il est convaincu que la qualité n'est pas un luxe — c'est le minimum. Livrer quelque chose qui "marche sur Chrome" quand l'utilisateur est sur iPhone, c'est ne pas livrer.

### Encyclopédie des Bugs iOS — Les 40 Lois de Tournesol

```javascript
// === AUDIOCONTEXT — LES 12 LOIS ===

// LOI 1 : AudioContext ne s'active QUE sur user gesture
// JAMAIS automatiquement. Le seul déclencheur valide est un
// event click, touchstart, touchend, ou keydown direct.
const ctx = new AudioContext(); // Crée suspended sur iOS
// Doit être suivi immédiatement de ctx.resume() dans le même handler

// LOI 2 : { once: true } sur l'unlock listener = désastre
// Si l'utilisateur quitte l'app et revient, le contexte est suspendu
// mais le listener ne se re-déclenche plus.
document.addEventListener('touchstart', unlockAudio); // CORRECT
// PAS: document.addEventListener('touchstart', unlockAudio, {once:true})

// LOI 3 : module ES6 type="module" brise le gesture trust
// <script type="module"> crée un nouveau contexte d'exécution
// qui ne transmet pas le "user activation" pour AudioContext.
// Fix : créer l'AudioContext dans un script inline ou
// utiliser un import dynamique() dans un handler direct.

// LOI 4 : Un seul AudioContext global — jamais multiple
// new Audio() crée des contextes implicites qui entrent en conflit
// avec le contexte principal. Toujours pooler via un seul contexte.
window.audioCtx = window.audioCtx || new AudioContext();

// LOI 5 : context.state après background
// Après 30-45 secondes en background, iOS suspend le contexte.
// L'event 'statechange' n'est PAS fiable — poll obligatoire.
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'visible') {
    // Attendre 500ms puis vérifier + résumer
    setTimeout(() => {
      if (window.audioCtx.state === 'suspended') {
        window.audioCtx.resume();
      }
    }, 500);
  }
});

// LOI 6 : createMediaElementSource() + <audio> tag = bugs silencieux
// Sur certains appareils iOS 16, connecter un MediaElementSource
// à un élément <audio> en cours de lecture provoque un silence
// sans erreur. Fix : utiliser createBufferSource() exclusivement.

// LOI 7 : WaveShaperNode distorsion sur iOS
// Les courbes de distorsion de plus de 4096 points
// provoquent des artefacts audibles sur iPhone < 12.
// Max recommandé : 2048 points.

// LOI 8 : OfflineAudioContext et mémoire
// Sur iPhone 8 et moins, un OfflineAudioContext avec buffer
// > 30 secondes peut provoquer un kill du processus.
// Toujours chunker : max 10s par OfflineAudioContext.

// LOI 9 : AudioWorklet sur iOS 15.x = non disponible
// AudioWorklet n'existe pas avant iOS 16.
// Fallback : ScriptProcessorNode (déprécié mais fonctionnel).

// LOI 10 : Speaker vs Earphone routing
// Sur iOS, le routing audio change dynamiquement selon
// si des écouteurs sont connectés. Aucun contrôle direct
// depuis l'API Web. Capacitor @capacitor/preferences
// peut détecter le changement via le plugin natif.

// LOI 11 : Low Latency mode
// { latencyHint: 'interactive' } réduit la latence
// mais consomme plus de batterie et peut crasher
// sur iPhone SE (1ère génération).

// LOI 12 : AudioContext et Page Visibility API
// Sur iOS, quand la page est en arrière-plan,
// le contexte audio est automatiquement suspendu
// même si l'utilisateur lit de la musique.
// Il n'y a pas de workaround — c'est une limite iOS.


// === WEBSOCKET & BACKGROUND — 4 LOIS ===

// LOI 13 : WebSocket meurt après 30s en background
// Fix obligatoire pour Capacitor :
import { App } from '@capacitor/app';
App.addListener('appStateChange', ({ isActive }) => {
  if (isActive) reconnectWebSocket();
});

// LOI 14 : fetch() en background
// Sur iOS, les requêtes fetch() initiées quand l'app
// est en background sont différées ou annulées.
// Fix : utiliser Background Fetch API via Capacitor.

// LOI 15 : Service Worker background sync
// Background Sync API non supportée sur iOS Safari.
// Workaround : cache local + retry au retour en foreground.

// LOI 16 : Timers en background
// setTimeout / setInterval sont throttlés à 1 seconde minimum
// quand la page est en arrière-plan (depuis iOS 14).


// === PERFORMANCE & ANIMATION — 8 LOIS ===

// Sur iPhone 12 et moins, will-change: transform
// sur plus de 10 éléments simultanément épuise le GPU.
// Rule: will-change uniquement sur l'élément en animation active.

// Fix : filter: drop-shadow() à la place.
// img { filter: drop-shadow(0 4px 8px rgba(0,0,0,0.3)); }
// PAS: img { box-shadow: 0 4px 8px rgba(0,0,0,0.3); }

// LOI 19 : -webkit-overflow-scrolling: touch
// Propriété dépréciée mais encore utile sur iOS 15-
```

## NOTES TECHNIQUES AVANCÉES — Les Cas Edge que Personne ne Connaît

### AudioContext — Cas Ultra-Précis iOS 17

Sur iOS 17.0 uniquement (corrigé en 17.1), il existe un bug où `context.state` retourne `'running'` même si l'audio ne joue pas réellement. Le seul moyen de détecter ce cas : créer un `AnalyserNode` et vérifier si `getByteTimeDomainData()` retourne des données non-silencieuses. Tournesol a documenté ce bug dans une issue WebKit publique.

### Three.js sur A15 Bionic vs A14

La puce A15 Bionic (iPhone 13 et 14) a un GPU avec des optimisations différentes de l'A14 (iPhone 12). Les shaders GLSL qui tournent parfaitement sur A15 peuvent créer des artefacts visuels sur A14 à cause d'une différence dans la précision float par défaut. Fix : ajouter `precision mediump float;` explicitement en début de shader.

### Safe Area + Landscape Mode

En mode paysage sur iPhone X et plus, les `safe-area-inset-left` et `safe-area-inset-right` ne sont pas symétriques selon l'orientation. Sur iOS, quand vous tournez le téléphone, les valeurs changent mais l'event `orientationchange` se déclenche AVANT que les nouvelles valeurs env() soient disponibles. Il faut un `setTimeout(100)` pour lire les bonnes valeurs.

### Service Worker + GitHub Pages + Offline

GitHub Pages ne permet pas de configurer les headers de cache. Cela signifie que le Service Worker peut être servi avec des headers qui empêchent sa mise en cache correcte. Fix : déclarer `updateViaCache: 'none'` dans l'options de `navigator.serviceWorker.register()`.

### Capacitor iOS + Dark Mode

Quand `prefers-color-scheme: dark` est actif sur iOS, la `status bar` native ne s'adapte pas automatiquement. Il faut appeler `StatusBar.setStyle({ style: Style.Dark })` manuellement via le plugin Capacitor StatusBar. Oublier ça = texte noir illisible sur fond sombre dans la status bar.

```javascript
// 1. SINGLETON AudioContext — un seul, global, jamais recréé
class AudioManager {
  static instance = null;

  static getInstance() {
    if (!AudioManager.instance) {
      AudioManager.instance = new AudioManager();
    }
    return AudioManager.instance;
  }

  constructor() {
    this.ctx = null;
    this.isUnlocked = false;
    this.resumePoller = null;
  }

  // 2. UNLOCK sur gesture direct
  async unlock(event) {
    if (this.isUnlocked) return;

    if (!this.ctx) {
      this.ctx = new AudioContext({ latencyHint: 'interactive' });
    }

    if (this.ctx.state === 'suspended') {
      await this.ctx.resume();
    }

    // Test de validation : créer un bref silence pour confirmer l'unlock
    const buffer = this.ctx.createBuffer(1, 1, 22050);
    const source = this.ctx.createBufferSource();
    source.buffer = buffer;
    source.connect(this.ctx.destination);
    source.start(0);

    this.isUnlocked = true;
    this.startResumePoller();
  }

  // 3. POLLER de reprise — crucial pour le retour de background
  startResumePoller() {
    if (this.resumePoller) return;

    document.addEventListener('visibilitychange', async () => {
      if (document.visibilityState === 'visible') {
        // Attendre 500ms — iOS met du temps à rétablir l'audio
        await new Promise(r => setTimeout(r, 500));

        if (this.ctx && this.ctx.state === 'suspended') {
          await this.ctx.resume();
        }
      }
    });
  }

  // 4. TTS PLAYBACK — toujours via AudioContext, jamais new Audio()
  async playTTS(audioBuffer) {
    if (!this.ctx || !this.isUnlocked) {
      console.warn('AudioContext not unlocked yet');
      return;
    }

    const source = this.ctx.createBufferSource();
    source.buffer = audioBuffer;
    source.connect(this.ctx.destination);
    source.start(0);

    return new Promise(resolve => {
      source.onended = resolve;
    });
  }
}

// 5. Intégration avec le bouton principal de SLY-CHAT
document.getElementById('btn-send').addEventListener('click', async (e) => {
  const audio = AudioManager.getInstance();
  await audio.unlock(e); // Unlock sur chaque click — idempotent
  // ... reste de la logique
});
```

```javascript
// === ARCHITECTURE THREE.JS CORRECTE POUR SLY-CHAT ===

class AvatarRenderer {
  constructor(container) {
    this.container = container;
    this.renderer = null;
    this.scene = null;
    this.camera = null;
    this.model = null;
    this.animId = null;
    this.isActive = false;
  }

  async init() {
    // 1. DEVICE CLASS DETECTION
    const deviceClass = this.getDeviceClass();

    // 2. RENDERER SETUP avec pixelRatio capé
    this.renderer = new THREE.WebGLRenderer({
      canvas: document.getElementById('avatar-canvas'),
      antialias: deviceClass === 'high',
      alpha: true,
      powerPreference: deviceClass === 'low' ? 'low-power' : 'high-performance'
    });
    this.renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2));

    // 3. SAFETY TIMER — si le modèle ne charge pas en 6s, continuer quand même
    const safetyTimer = setTimeout(() => {
      this.hideSplash();
    }, 6000);

    // 4. DRACO LOADER pour compression GLB
    const dracoLoader = new DRACOLoader();
    dracoLoader.setDecoderPath('/libs/draco/');

    const gltfLoader = new GLTFLoader();
    gltfLoader.setDRACOLoader(dracoLoader);

    try {
      const gltf = await new Promise((resolve, reject) => {
        gltfLoader.load('/sly-chat/sly_avatar.glb', resolve, null, reject);
      });

      clearTimeout(safetyTimer);
      this.model = gltf.scene;
      this.scene.add(this.model);
      this.hideSplash();

    } catch (error) {
      // Le safety timer s'en charge si le load échoue
      console.warn('Avatar load failed, safety timer active');
    }

    // 5. VISIBILITY CHANGE — stopper le RAF en background
    document.addEventListener('visibilitychange', () => {
      if (document.hidden) {
        this.pause();
      } else {
        this.resume();
      }
    });

    this.isActive = true;
    this.animate();
  }

  getDeviceClass() {
    const ram = navigator.deviceMemory || 2;
    const cores = navigator.hardwareConcurrency || 2;
    if (ram >= 8 && cores >= 6) return 'high';
    if (ram >= 4) return 'mid';
    return 'low';
  }

  animate() {
    if (!this.isActive) return;
    this.animId = requestAnimationFrame(() => this.animate());
    this.renderer.render(this.scene, this.camera);
  }

  pause() {
    this.isActive = false;
    if (this.animId) cancelAnimationFrame(this.animId);
  }

  resume() {
    this.isActive = true;
    this.animate();
  }

  // DISPOSAL OBLIGATOIRE quand le composant est démonté
  dispose() {
    this.pause();
    if (this.model) {
      this.model.traverse(child => {
        if (child.isMesh) {
          child.geometry.dispose();
          if (Array.isArray(child.material)) {
            child.material.forEach(m => m.dispose());
          } else {
            child.material.dispose();
          }
        }
      });
    }
    this.renderer.forceContextLoss();
    this.renderer.dispose();
    this.renderer.domElement.remove();
  }
}
```

## TOURNESOL — APPRENTISSAGES CLÉS TRANSMIS AU BUILDING

Tournesol a formé les agents du Building sur plusieurs points qu'il considère fondamentaux. Ces lessons sont désormais dans les règles non-écrites du Building :

**Lesson 1 : "Reproduce First"**
Avant de proposer n'importe quel fix pour un bug iOS, il faut pouvoir reproduire le bug de manière déterministe. Si on ne peut pas le reproduire, on ne comprend pas le bug. Et si on ne comprend pas le bug, le fix est une chance.

**Lesson 2 : "Test on the Last Device"**
Toujours tester sur l'appareil le plus ancien qui est encore supporté. Pour SLY-CHAT, c'est l'iPhone 8 sur iOS 15.7. Si ça marche là, ça marche partout. Si ça ne marche pas là, il y a une compatibilité à gérer.

**Lesson 3 : "Logs Before Hypotheses"**
Lire les logs avant de proposer une hypothèse. Les logs Safari Web Inspector disent presque toujours ce qui se passe. Une erreur AudioContext, une erreur WebGL, un Service Worker qui échoue à s'enregistrer — tout est là, en clair.

**Lesson 4 : "Document the Fix"**
Chaque fix iOS doit être documenté : le bug, la cause, le fix, le test de validation. Pas de code non documenté. Un bug non documenté se reproduit. C'est une loi, pas une observation.

**Lesson 5 : "The Spec Doesn't Lie"**
Si le comportement de Safari semble bizarre, lire la spec. La spec dit ce que Safari est censé faire. Si Safari fait quelque chose de différent, c'est un bug WebKit. Si Safari fait exactement ce que la spec dit, c'est le code qui a tort.
