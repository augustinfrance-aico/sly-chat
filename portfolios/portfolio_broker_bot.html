Approche Operationnelle — 5 Phases
1

Decouverte & Configuration

Jours 1-2
  • Cartographier le flux d'inscription courtier (parcours EU vs Non-EU)
  • Definir les scripts de conversation & cas limites
  • Configurer le Bot Telegram via BotFather
  • Mettre en place le dashboard (respond.io ou Chatwoot)
  • Configurer les roles equipe & permissions
2

Construction du Moteur IA

Jours 3-5
  • Construire le prompt IA personnalise avec la base de connaissances
  • Detection & routage EU/Non-EU
  • Gestion d'etat de conversation (ou en est l'utilisateur dans le funnel)
  • Detection de fallback — quand l'IA ne peut pas gerer, signaler pour humain
  • Tester tous les parcours de conversation de bout en bout
3

Workflows d'Automatisation

Jours 5-7
  • Workflows n8n : routage messages, tagging, mises a jour statut
  • Auto-tag utilisateurs : EU / Non-EU / Inscrit / Besoin Humain
  • Declencheurs de notification pour reprise humaine
  • Journalisation conversations & suivi style CRM
  • Endpoints webhook pour tous les evenements
4

Dashboard & Configuration Equipe

Jours 7-8
  • Connecter le bot Telegram au dashboard
  • Configurer les comptes equipe & permissions par role
  • Admin : acces complet. Agents : reponse uniquement. Lecteurs : lecture seule
  • Workflow de reprise humaine — transition fluide IA → humain
  • Capacite de reponse manuelle a tout moment de la conversation
5

Tests & Lancement

Jours 9-10
  • Tests complets de bout en bout : parcours EU, parcours Non-EU, reprise humaine
  • Tests de charge — gerer les conversations simultanees
  • Formation de l'equipe
  • Mise en production + 48h de surveillance
