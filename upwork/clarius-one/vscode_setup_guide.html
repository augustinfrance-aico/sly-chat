Quest 2 — Install

Install the Required Tools

Get Node.js and VSCode ready
+100 XP
▼

Claude Code needs two things to work: VSCode (your code editor) and Node.js (a program that runs in the background).

Install Node.js

Go to the website below, click the big green "LTS" button, and install it like any normal program.

website Open → 
Download Node.js from:
https://nodejs.org

Click the green "LTS" button (recommended version).
Install with all default settings. Just click Next → Next → Finish.

Verify Node.js is installed

Open a terminal (or Command Prompt on Windows) and type this command:

terminal 
node --version
✅

If you see something like v22.x.x — you're good! Any version 18 or higher works.

⚠️

Don't see it? Close your terminal, restart VSCode, and try again. On Windows, you might need to restart your computer after installing Node.js.

VSCode is installed and open

If you don't have VSCode yet, download it from code.visualstudio.com

Quest 4 — Directives

Set Up Your Project Rules

Tell Claude how you want things done
+150 XP
▼
🎯

This is the most important step. Claude reads a file called CLAUDE.md at the root of your project EVERY time you start a conversation. It's like giving a new employee their job manual on day one.

Open your project folder

In VSCode: File → Open Folder → pick your project's root folder. Claude works at the folder level, not file level.

Create CLAUDE.md at the root of your project

Right-click in the file explorer → New File → name it CLAUDE.md

Here's a simple template you can start with. Copy-paste it and customize:

CLAUDE.md 
# My Project

## What is this project?
A brief description of what your app/project does.

## Tech Stack
- List your technologies here (React, Node.js, Python, etc.)
- List your database (Supabase, Firebase, PostgreSQL, etc.)
- List your hosting (Vercel, Netlify, Railway, etc.)

## Rules for Claude
- Always write clean, readable code
- Add comments to explain complex logic
- Never expose API keys in the code
- Ask before deleting any file

## Project Structure
- src/          → source code
- public/       → static files
- package.json  → dependencies

## Important Notes
- Any special instructions go here
- Things Claude should know about your project
✨

You already wrote your directives! You're ahead of the game. Just make sure they're in a file called CLAUDE.md at the root of your project folder. If your directives are in a different file, simply copy the content into CLAUDE.md.

CLAUDE.md exists with my rules inside

Check that CLAUDE.md is at the root of your project (same level as package.json or your main files).

🔒 Keep your secrets safe

🚫

NEVER put passwords, API keys, or secrets in CLAUDE.md. Those go in a .env file which is NOT shared or committed to GitHub. Make sure your .gitignore file includes .env.

My .env file has my secrets (not CLAUDE.md)

API keys, database URLs, passwords → all in .env, never in CLAUDE.md.

Quest 5 — First Chat

Talk to Claude for the First Time

Send your first messages and see the magic
+100 XP
▼

Time to try it out! Open the Claude Code panel and type a message. Start simple.

Ask Claude to describe your project

Type: "What is this project about? What files do you see?"

🤯

Claude will read your CLAUDE.md + scan your files and give you a summary. This proves it can see your project. If it describes your project correctly — your setup is working!

Ask Claude to do something small

Type: "Add a comment at the top of my main file saying 'Hello from Claude'"

👀

Claude will ask for permission before editing the file. Read what it wants to change, then click "Allow" if it looks good. This is how it works every time — you always have the final say.

Try a bigger request

Type something real: "Explain the structure of this project and suggest 3 improvements"

📝 Example things you can say

messages you can try 
--- BUILDING ---
"Create a contact form with name, email, and message fields"
"Add a dark mode toggle to the header"
"Set up a new API endpoint for user profiles"

--- FIXING ---

"Explain this function to me like I'm a beginner"

--- MAINTAINING ---
"Run the tests and tell me if anything fails"
"Create a git commit with all my changes"

🧠 Pro Tips

Be specific, not vague

❌ "Fix the page" → ✅ "The login page shows a blank screen after clicking Submit — fix it"

One task per message

Don't ask for 5 things at once. Break big tasks into small steps. Claude works best with focus.

Always review before approving

Read what Claude wants to change before clicking "Allow". It's fast but your code, your rules.

Update CLAUDE.md as you go

Every new convention or rule? Add it to CLAUDE.md. Future sessions will be smarter automatically.

Say "explain" when lost

After any change, type "explain what you just did and why" — Claude will break it down for you.
